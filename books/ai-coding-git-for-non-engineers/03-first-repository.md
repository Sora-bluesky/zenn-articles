## この章のゴール

> この章を終えると、VS Code の画面操作だけでリポジトリ（プロジェクトの箱）を作成でき、AI ツールの「作業場所」を用意できるようになる。

## リポジトリとは何か

AI ツールに仕事を頼むには、まず **作業場所** が必要である。

Git では、この作業場所のことを **リポジトリ（repository）** と呼ぶ。日本語に訳すと「貯蔵庫」だが、本書では **「プロジェクトの箱」** と呼ぶことにする。

この「箱」の中に、ファイルを入れたり、変更を記録したりしていく。

```
プロジェクトの箱（リポジトリ）
├── index.html     ← AIが作ったファイル
├── style.css      ← AIが作ったファイル
└── .git/          ← 変更履歴を記録する場所（Gitが自動管理）
```

`.git` というフォルダが、Git が変更履歴を記録している場所である。中身を触る必要はない。「箱の中にある、記録帳」くらいに思っておけばよい。

## Git の「3 つのエリア」を理解する

Git でファイルを管理するとき、ファイルは **3 つのエリア** を移動する。この仕組みは、本書の[第 4 章](04-add-and-commit)以降でずっと使う最重要概念である。

![Git の3つのエリア](/images/book-03-three-areas.png)

| エリア                 | 日常の例え     | 何が置かれるか                       |
| ---------------------- | -------------- | ------------------------------------ |
| **作業ディレクトリ**   | 机の上         | 今まさに編集しているファイル         |
| **ステージングエリア** | 発送カウンター | 「次に記録に含める」と決めたファイル |
| **リポジトリ**         | 倉庫の棚       | 確定済みの記録（コミット）           |

:::message
### 別の例え: 写真アルバム作り
- **作業ディレクトリ** = カメラロール（撮りためた写真がある場所）
- **ステージング** = 「このアルバムに入れる」と選んだ写真
- **リポジトリ** = 完成したアルバム（もう変わらない記録）

写真をたくさん撮っても（ファイルを編集しても）、アルバムに入れる写真を選んで（add）、アルバムに貼り付けて（commit）初めて記録になる。
:::

今の段階で完全に理解しなくて構わない。「ファイルは 3 段階を経て記録される」ということだけ頭に入れておいてほしい。[第 4 章](04-add-and-commit)で実際に操作する。

## VS Code でリポジトリを作る

### 手順 1: プロジェクト用のフォルダを作る

まず、パソコン上にプロジェクト用のフォルダを作る。

**Windows の場合:**

1. エクスプローラーを開く
2. 「ドキュメント」フォルダに移動
3. 右クリック →「新規作成」→「フォルダー」
4. フォルダ名を `my-first-project` にする

**Mac の場合:**

1. Finder を開く
2. 「書類」フォルダに移動
3. 右クリック →「新規フォルダ」
4. フォルダ名を `my-first-project` にする

**Linux の場合:**

1. ファイルマネージャーで「ドキュメント」フォルダを開く
2. 右クリック →「新しいフォルダー」
3. 名前を `my-first-project` にする

ターミナルからの場合:

```bash
mkdir ~/Documents/my-first-project
```

### 手順 2: VS Code でフォルダを開く

1. VS Code を起動
2. メニューの「ファイル」→「フォルダーを開く」をクリック
3. 先ほど作った `my-first-project` フォルダを選択

VS Code の画面左側に、空のフォルダが表示される。

### 手順 3: リポジトリを初期化する

1. VS Code の左側メニューから、分岐のアイコン（ソース管理）をクリック
2. 「リポジトリを初期化する」ボタンが表示される
3. このボタンをクリック

これで、フォルダの中に `.git` という隠しフォルダが作られ、リポジトリ（プロジェクトの箱）が完成する。

:::message
ボタンをクリックしただけで、裏では `git init` というコマンドが実行されている。VS Code が代わりにやってくれているのである。
:::

### 手順 4: 最初のファイルを作る

箱を作っただけでは中身が空なので、テスト用のファイルを 1 つ作ってみよう。

1. VS Code の左側「エクスプローラー」パネルで、フォルダ名の横にある「新しいファイル」アイコンをクリック
2. ファイル名を `hello.txt` にする
3. ファイルが開くので、何か文字を入力する（例: 「はじめてのGit」）
4. `Ctrl + S`（Mac は `Cmd + S`）で保存

### 手順 5: 変更を確認する

1. 左側の「ソース管理」アイコンをクリック
2. 「変更」セクションに `hello.txt` が表示されている

ファイル名の横に **U** という文字が見えるはずだ。これは **Untracked**（まだ追跡されていない）という意味で、「Git がまだこのファイルの存在を知らない」状態を表している。

### ファイルの状態を読む — VS Code のステータス記号

VS Code の「ソース管理」パネルには、ファイルの状態を示すアルファベットが表示される。AI コーディングをしていると、AI が大量のファイルを変更するので、この記号を読めると「AI が何をしたか」が一目でわかる。

| 記号  | 英語      | 意味                                                 |
| ----- | --------- | ---------------------------------------------------- |
| **U** | Untracked | Git がまだ知らない新しいファイル                     |
| **A** | Added     | ステージング済み（次の記録に含まれる）の新規ファイル |
| **M** | Modified  | Git が追跡中のファイルが変更された                   |
| **D** | Deleted   | ファイルが削除された                                 |
| **R** | Renamed   | ファイル名が変更された                               |
| **C** | Conflict  | マージでコンフリクト（競合）が発生した               |

今の段階では **U**（新しいファイル）と **M**（変更済み）だけ覚えておけば十分。

## .git フォルダの正体 — 触らなくていいが知っておくと安心

リポジトリを初期化すると作られる `.git` フォルダの中身を少しだけ覗いてみよう。 **触る必要はまったくない** が、中に何があるか知っていると「Git は魔法ではない」ことがわかって安心する。

```
.git/
├── HEAD          ← 今どのブランチにいるか（第6章で詳しく扱う）
├── config        ← このリポジトリの設定
├── objects/      ← ファイルの中身やコミット情報の実体
├── refs/         ← ブランチやタグの参照先
└── hooks/        ← 自動実行スクリプト（上級者向け）
```

| フォルダ/ファイル | 役割                               | 触る必要                        |
| ----------------- | ---------------------------------- | ------------------------------- |
| `HEAD`            | 「今いるブランチ」を記録           | なし                            |
| `config`          | リモート URL 等の設定              | 通常なし（`git config` で設定） |
| `objects/`        | すべてのファイル内容とコミット情報 | **絶対に触らない**              |
| `refs/`           | ブランチ名→コミットの対応表        | なし                            |

要するに、 **`.git` フォルダは Git の「脳」** である。ここが壊れると履歴がすべて失われるので、直接触るのは避けること。

次の章で、このファイルの変更を Git に「記録」する方法を学ぶ。

## 「やらかした！」復旧コーナー

### 「間違ったフォルダでリポジトリを初期化してしまった」

心配無用。リポジトリの初期化は「.git フォルダを作っただけ」なので、取り消すには .git フォルダを削除すればよい。

ただし、.git フォルダは隠しフォルダなので、通常のファイル操作では見えない。

:::message alert
**注意**: この操作で Git の記録がすべて消える。初期化直後にのみ使うこと。
:::

VS Code のターミナルで以下を実行する。

**Windows（PowerShell）の場合:**

```powershell
Remove-Item -Recurse -Force .git
```

**Mac / Linux の場合:**

```bash
rm -rf .git
```

## 裏で何が起きているか（CLI 解説）

この章で VS Code が裏で実行したコマンドは 1 つだけ。

| 操作                             | CLI コマンド | 意味                             |
| -------------------------------- | ------------ | -------------------------------- |
| 「リポジトリを初期化する」ボタン | `git init`   | 現在のフォルダをリポジトリにする |

リポジトリの状態を確認するコマンドも覚えておくと便利である。

```bash
git status
```

これを実行すると、現在のブランチ名、ステージングされたファイル、変更されたファイル、まだ追跡されていないファイルが一覧で表示される。AI コーディング中に「今どうなっている？」と確認したくなったら、まずこのコマンドを実行するとよい。

実際の表示例:

```
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        hello.txt

nothing added to commit but untracked files present (use "git add" to track)
```

この表示は「main ブランチにいて、hello.txt というまだ追跡されていないファイルがある」と教えてくれている。`git status` は何も変更しない安全なコマンドなので、不安なときは何度でも実行して構わない。

`git status` の出力を1行ずつ読んでみよう。Git があなたに「今の状態」を教えてくれている。

```
On branch main               ← 今いるブランチの名前
No commits yet                ← まだ1回も記録していない
Untracked files:              ← Git がまだ知らないファイルがある
  (use "git add <file>..."    ← 「git add で追跡を始められるよ」と教えてくれている
   to include in what will be committed)
        hello.txt             ← 該当するファイル名
```

英語のメッセージが並ぶと怖く感じるが、 **Git は「今どういう状態で、次に何をすればいいか」を毎回教えてくれている**。エラーが出ても、メッセージをよく読めばヒントが書いてある。

短縮表示にしたい場合は `-s`（short）オプションを使う。

```bash
git status -s
```

```
?? hello.txt
```

`??` は「Untracked（未追跡）」の意味である。前のセクションで説明した記号と対応している。

次の[第 4 章](04-add-and-commit)で、このファイルを実際に Git に記録する。ここまでできていれば、あと2クリックで完了だ。

## 確認クイズ

- Q1: 「リポジトリ」を一言で言うと何？ → A: プロジェクトの箱（変更履歴を記録する場所）
- Q2: `git status` で表示される「U」は何の略？ → A: Untracked（まだ追跡されていない）
- Q3: `.git` フォルダを削除するとどうなる？ → A: Git の記録がすべて消える（ファイル自体は残る）
