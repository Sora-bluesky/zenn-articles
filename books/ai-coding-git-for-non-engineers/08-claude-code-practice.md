## この章のゴール

> この章を終えると、Claude Code を使って実際にアプリを作り、AI が行う Git 操作の意味が理解できるようになる。

:::message
### コラム: Vibe Coding の落とし穴
「Vibe Coding」とは、AI に自然言語で指示してコードを書かせる開発スタイルのこと。本書の Claude Code を使った開発は、まさに Vibe Coding の一形態。

Git なしで Vibe Coding を始めると、こんな問題が起きる:

1. **元に戻せない** — AI が変なコードを書いても、前の状態に戻す手段がない
2. **何が変わったかわからない** — AI がどのファイルをどう書き換えたか追跡できない
3. **公開できない** — 完成したものを GitHub Pages で公開するには Git が必要
4. **共有できない** — 「見て」と誰かに頼むには GitHub の URL が一番簡単

本書を読み終えた皆さんは、すでに「安全な Vibe Coding」の基盤を持っている。
:::

## Claude Code とは

Claude Code は、Anthropic が提供する **AI コーディングツール** である。ターミナル（文字入力画面）に日本語で指示を出すと、AI がコードを書き、ファイルを作成・編集し、Git 操作まで自動でやってくれる。

特徴をまとめると以下の通り。

| 特徴       | 内容                                                       |
| ---------- | ---------------------------------------------------------- |
| 入力方法   | ターミナルに日本語で指示                                   |
| できること | ファイル作成・編集・削除、Git 操作、テスト実行             |
| 動作場所   | 自分の PC 上（ローカル）                                   |
| 必要なもの | Git、GitHub アカウント                                     |
| 料金       | Anthropic API の従量課金（Max プランなら月額定額で利用可） |

## Claude Code をインストールする

VS Code のターミナルで以下を実行する。

**Windows（PowerShell）の場合:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Mac / Linux の場合:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

:::message
`irm` は `Invoke-RestMethod`（Web からデータを取得するコマンド）、`iex` は `Invoke-Expression`（取得した内容を実行するコマンド）の省略形である。Mac / Linux の `curl | bash` も同じ仕組みで、インストールスクリプトをダウンロードして実行している。
:::

:::message alert
Windows では **Git for Windows** が必要である。第 2 章でインストール済みなら問題ない。
:::

インストール完了後、以下で確認する。

```bash
claude --version
```

バージョン番号が表示されれば OK。ネイティブインストーラーは自動更新に対応しているため、常に最新版が使える。

### 初回認証

初めて Claude Code を起動すると、Anthropic アカウントでの認証が求められる。

```bash
claude
```

ブラウザが開くので、画面の指示に従ってログインする。

## 最初のプロジェクト: 自己紹介ページを作る

### 手順 1: リポジトリを準備する

第 3 章〜第 5 章で学んだ手順で、GitHub リポジトリを用意する。

1. パソコンに新しいフォルダ `my-profile-site` を作成
2. VS Code でフォルダを開く
3. 「ソース管理」→「リポジトリを初期化する」
4. GitHub にリポジトリ `my-profile-site` を作成
5. リモートを登録して push

```bash
git remote add origin https://github.com/ユーザー名/my-profile-site.git
git push -u origin main
```

### 手順 2: Claude Code を起動する

VS Code のターミナルで、プロジェクトフォルダ内にいることを確認してから起動する。

```bash
claude
```

Claude Code のインターフェースが表示される。ここに日本語で指示を入力する。

### 手順 3: AI に指示を出す

以下のように入力してみよう。

```
自己紹介ページを作ってください。
名前: 山田太郎
趣味: 読書、カメラ
職業: デザイナー
シンプルなHTML + CSSで、レスポンシブ対応にしてください。
```

Enter を押すと、Claude Code が作業を開始する。

### 手順 4: AI の作業を観察する

Claude Code は、以下のような手順で作業を進める。

```
Claude Code: 自己紹介ページを作成します。

1. index.html を作成しています...
2. style.css を作成しています...

ファイルを作成しました。変更を確認してください。
```

ここで重要なのは、 **Claude Code がファイルを作成した後、あなたに確認を求めている** 点である。AI は勝手にコミットせず、あなたの承認を待つ。

### 手順 5: 変更を確認する

VS Code の「ソース管理」パネルを開くと、AI が作成したファイルが一覧で見える。

ファイルをクリックすると、 **差分ビュー**（変更前と変更後の比較画面）が開く。新規ファイルの場合はすべて緑色（追加）で表示される。

内容を確認して問題なければ、第 4 章で学んだ手順でコミットする。

1. **+** ボタンで全ファイルをステージング
2. コミットメッセージを入力（例: `feat: 自己紹介ページを作成`）
3. **✓** ボタンでコミット

### 手順 6: GitHub に公開する

```bash
git push
```

GitHub のリポジトリページでファイルが表示されていれば成功。

## 試してみよう: コピペで始める3つのプロジェクト

自己紹介ページ以外にも、以下のプロンプトをそのままコピペして試せる。

### 2. デジタル名刺（デザイン寄り）

### 3. TODO リスト（インタラクティブ）


## CLAUDE.md — AI にプロジェクトのルールを教える

Claude Code にはプロジェクトの文脈を伝える仕組みがある。プロジェクトのルートに **`CLAUDE.md`** というファイルを置くと、Claude Code が起動時にそれを読み込み、指示に従ってくれる。

### CLAUDE.md に書く内容の例

```markdown
# プロジェクト概要

個人のポートフォリオサイト。HTML + CSS のみで構成。

# ルール

- 日本語で作成する
- レスポンシブデザインにする
- 外部ライブラリは使わない
- コミットメッセージは日本語で書く

# ファイル構成

- index.html: トップページ
- about.html: 自己紹介ページ
- style.css: スタイルシート
```

この `CLAUDE.md` を書いておくだけで、Claude Code の出力が格段に意図通りになる。 **AI への「引き継ぎ書」** のようなものだと考えるとわかりやすい。

:::message
`CLAUDE.md` 自体もリポジトリに含まれるので、git add → commit しておくこと。GitHub 上にも公開される点に注意。
:::

## AI への効果的な指示の出し方

Claude Code に仕事を頼むとき、「何を言うか」で結果の質が大きく変わる。

### 良い指示の例

| 良い指示                                                                                             | なぜ良いか                   |
| ---------------------------------------------------------------------------------------------------- | ---------------------------- |
| 「トップページにヘッダーとフッターを追加して。ヘッダーにはサイト名、フッターにはコピーライトを表示」 | 具体的で、何を作るかが明確   |
| 「style.css のフォントサイズを 16px に統一して」                                                     | 対象ファイルと変更内容が明確 |
| 「about.html にプロフィール画像を追加して。画像は images/photo.jpg を使って」                        | ファイルパスまで指定している |

### 悪い指示の例

| 悪い指示           | なぜ悪いか                 |
| ------------------ | -------------------------- |
| 「いい感じにして」 | 何をすればいいかわからない |
| 「全部作って」     | 範囲が広すぎて AI が迷う   |
| 「バグを直して」   | どのバグかわからない       |

### コツ: 小さく頼んで、確認してから次へ

AI に「アプリ全体を一度に作って」と頼むと、大量のファイルが生成されて確認が大変になる。代わりに:

1. 「まずトップページだけ作って」
2. 確認 → コミット
3. 「次に問い合わせフォームを追加して」
4. 確認 → コミット

このように **小さな単位で頼み、その都度コミットする** のが安全である。Git のコミットが細かく分かれていれば、問題が起きた箇所まで戻すのも簡単。

## AI が裏で行う Git 操作を理解する

Claude Code を使っていると、AI が Git コマンドを実行する場面がある。これまでの章で学んだ知識があれば、何をしているか理解できるはずだ。

| AI の動作                 | Git 操作                           | 学んだ章 |
| ------------------------- | ---------------------------------- | -------- |
| ファイルを作成・編集する  | （Git 操作なし、ファイル操作のみ） | —        |
| 変更を記録に含める        | `git add`                          | 第 4 章  |
| 変更を確定する            | `git commit`                       | 第 4 章  |
| GitHub にアップロードする | `git push`                         | 第 5 章  |
| 新しいブランチで作業する  | `git switch -c`                    | 第 6 章  |
| Pull Request を作成する   | `gh pr create`                     | 第 7 章  |

## Codex・Antigravity・Copilot との使い分け

Claude Code 以外にも AI コーディングツールがある。それぞれの特徴を簡単にまとめる。

| ツール                 | 得意なこと                               | 使い方                     | Git の関わり              |
| ---------------------- | ---------------------------------------- | -------------------------- | ------------------------- |
| **Claude Code**        | ゼロからのアプリ作成、大規模な変更       | ターミナルに日本語で指示   | 自動で add/commit/push/PR |
| **Codex**              | 複数ファイルの一括生成、リファクタリング | Web UI またはターミナル    | 自動で commit/PR          |
| **Google Antigravity** | ゼロからのアプリ作成、リサーチ、コード生成 | ターミナルに日本語で指示   | 自動で add/commit/push    |
| **GitHub Copilot**     | コード補完、1 ファイル内の修正           | VS Code のエディタ上で提案 | ユーザーが手動で commit   |

### 使い分けの指針

- **新しいプロジェクトを作りたい** → Claude Code
- **Google のエコシステムを活用したい** → Google Antigravity
- **既存のコードを少し直したい** → GitHub Copilot
- **まとまった変更を一括で行いたい** → Codex

どのツールを使っても、Git の基本操作（commit、push、PR）は共通である。本書で学んだ知識がそのまま活きる。

## 「やらかした！」復旧コーナー

### 「AI が変なコードをコミットしてしまった」

AI の変更を取り消すには、 **revert（リバート）** を使う。

```bash
git revert HEAD
```

これは「直前のコミットを打ち消す新しいコミット」を作成するコマンドである。元のコミットは履歴に残るが、変更内容は元に戻る。

:::message
`git reset` でもコミットを取り消せるが、revert のほうが安全である。reset は履歴自体を書き換えるため、すでに push 済みのコミットに使うとトラブルの原因になる。
:::

### 「AI が大量のファイルを変更したので、全部やり直したい」

まだコミットしていない場合は、以下で全変更を取り消せる。

```bash
git restore .
```

これは「すべてのファイルを直前のコミット時点の状態に戻す」コマンドである。

:::message alert
`git restore .` は **コミットしていない変更をすべて捨てる** 操作である。ステージングしていない変更は完全に消えるので、慎重に使うこと。
:::

### 「AI の変更を一部だけ採用して、残りは捨てたい」

VS Code の差分ビューで、ファイルごとに判断できる。

1. 「ソース管理」パネルで変更ファイルの一覧を確認
2. 採用したいファイルだけ **+** ボタンでステージング
3. 採用しないファイルは右クリック →「変更を破棄」
4. ステージングしたファイルだけコミット

## 裏で何が起きているか（CLI 解説）

| 操作                       | CLI コマンド             | 意味                               |
| -------------------------- | ------------------------ | ---------------------------------- |
| 直前のコミットを打ち消す   | `git revert HEAD`        | 安全にコミットを取り消す           |
| コミット前の変更を全部戻す | `git restore .`          | ファイルを直前のコミット状態に復元 |
| 特定ファイルだけ戻す       | `git restore ファイル名` | そのファイルだけ復元               |
| コミット履歴を確認         | `git log --oneline`      | 記録の一覧を表示                   |
| 変更差分を確認             | `git diff`               | コミット前の変更内容を表示         |
