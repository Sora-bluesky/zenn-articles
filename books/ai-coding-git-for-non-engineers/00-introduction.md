## この本は誰のために書いたか

「AIにコードを書いてもらいたい」と思ったことはないだろうか。

Claude Code、Codex、Google Antigravity、GitHub Copilot ── 2026年現在、AIにコードを書かせるツールは驚くほど充実している。プログラミングの知識がなくても、日本語で「こういうアプリを作って」と頼めば、AIがコードを生成してくれる時代になった。

ところが、いざ使い始めようとすると、こんな壁にぶつかる。

> 「Git? リポジトリ? コミット? ……何のことかさっぱりわからない」

実は、これらのAIツールは **すべて Git という仕組みの上で動いている**。Git を知らないと、ツールのインストールすらままならない。

この本は、そんな「AIツールを使いたいのに、Git がわからなくて詰んでいる」人のために書いた。

## 対象読者

- プログラミング経験がゼロの人
- Claude Code / Codex / Google Antigravity / GitHub Copilot に興味がある人
- デザイナー、ライター、PM、ディレクター、個人事業主
- 「ターミナル」「コマンドライン」と聞くだけで不安になる人

**エンジニア向けの「Git の仕組みを深く理解する」本ではない。** あくまで「AIツールを動かすために最低限必要な Git 操作を、最短距離で身につける」ための本である。

## この本のゴール

本書を最後まで読み終えると、以下のことができるようになる。

1. **VS Code の画面操作だけ** で Git の基本操作（記録・公開・枝分かれ）ができる
2. **GitHub** にプロジェクトを公開し、他の人と共有できる
3. **Claude Code** や **Google Antigravity** を使って、AIに実際のアプリを作ってもらえる
4. 「やらかした！」ときに、自分で元に戻せる

## 前提条件

必要なものは3つだけ。

- **PC**（Windows、Mac、または Linux）
- **ブラウザ**（Chrome、Edge、Safari など）
- **やる気**

それ以外の知識は一切不要である。

## この本の読み方

### GUI（画面操作）を先、CLI（コマンド入力）を後に

世の中の Git 入門記事の多くは、黒い画面（ターミナル）にコマンドを打ち込むところから始まる。しかし、初めて Git に触れる人にとって、あの黒い画面はそれだけで心理的なハードルになる。

本書では、 **VS Code というエディタの画面上のボタン操作** で Git を扱う。各章の末尾に「裏で何が起きているか（CLI 解説）」を設けているので、慣れてきたらコマンドも覚えていけばよい。

### 「やらかした！」復旧コーナー

Git で一番怖いのは「取り返しのつかないことをしてしまうのではないか」という不安である。

安心してほしい。 **Git で本当に取り返しのつかないことは、ほとんど起きない。**

各章の末尾に「やらかした！」復旧コーナーを設けている。間違えたときの戻し方を知っていれば、怖いものはない。
