## この章のゴール

> この章を終えると、AIツールがなぜ Git を必要とするか理解でき、「Git を学ぶ理由」が自分の言葉で説明できるようになる。

## プログラミングを知らなくてもコードが書ける時代

2026年、AIコーディングツールは爆発的に広がっている。

- **Claude Code** ── ターミナル（文字入力画面）で「こういうアプリを作って」と日本語で頼むと、ファイルの作成からテストまで全部やってくれる
- **Codex** ── OpenAI が提供する AI コーディングツール。指示を出すと、複数のファイルにまたがるコードを一括で生成する
- **Google Antigravity** ── Google が提供するターミナルベースの AI コーディングツール。Gemini がコードを生成し、ファイル作成から Git 操作まで自動で行う
- **GitHub Copilot** ── VS Code のエディタ上でコードを書いていると、続きを予測して提案してくれるアシスタント

これらのツールを使えば、プログラミング言語を知らなくても「自分のウェブサイト」や「業務効率化ツール」を作ることができる。

実際に、デザイナーが自分のポートフォリオサイトを Claude Code や Antigravity で作ったり、営業担当者が顧客管理ツールを Copilot の力を借りて作ったりする事例が増えている。

## 「でも全部、Git が必要」

ここで残念なお知らせがある。

**これらのAIツールは、すべて Git という仕組みの上で動いている。**

なぜか。AIツールがコードを書くとき、裏側では次のようなことが起きている。

1. AIがコードを生成する
2. そのコードを **ファイルとして保存** する
3. 変更内容を **記録（コミット）** する
4. 必要に応じて **インターネット上に公開（プッシュ）** する

この 2〜4 のステップが、まさに Git の仕事である。

Claude Code を例に見てみよう。Claude Code に「自己紹介ページを作って」と頼むと、こんなことが起きる。

```
あなた: 自己紹介ページを作って。名前は山田太郎、趣味は読書。

Claude Code: わかりました。以下のファイルを作成します。
  - index.html（ページ本体）
  - style.css（デザイン）

  ファイルを作成しました。
  git add で変更をステージングし、git commit で記録します。
```

この「git add」「git commit」が何をしているか、今の時点ではわからなくて構わない。重要なのは、 **AI ツールが自動的に Git を使っている** という事実である。

## Git とは何か — 3行で理解する

Git を一言で説明するなら、 **「ファイルの変更履歴を記録するしくみ」** である。

もう少し噛み砕くと、こういうことだ。

| ふだんの作業               | Git を使った場合                 |
| -------------------------- | -------------------------------- |
| ファイルを上書き保存する   | 変更内容を **記録つきで保存** する |
| 「元に戻す」は Ctrl+Z だけ | **いつでも過去の状態に戻せる**   |
| どこを変えたか忘れる       | **何をいつ変えたか全部残る**     |

Word や Google ドキュメントの「変更履歴」機能を想像してほしい。あれの超強力版が Git である。

### Git と GitHub の違い

初心者が最初に混乱するポイントがここだ。

- **Git** = パソコンの中で動く「変更履歴の記録ソフト」
- **GitHub** = 記録したものを **インターネット上に保管・共有するサービス**

例えるなら、Git は「日記帳」、GitHub は「日記帳を保管するクラウドの本棚」である。

## なぜ「非エンジニア」に Git が必要になったのか

かつて Git はエンジニアだけのツールだった。しかし、AI コーディングツールの登場で状況が変わった。

### 理由 1: AI ツールが Git を前提としている

前述の通り、Claude Code も Codex も Antigravity も Copilot も、Git がインストールされていないと動かない。AI に仕事を頼むための「作業場所」が Git リポジトリ（プロジェクトの箱）なのである。

### 理由 2: 「AI が書いたコード」を管理する必要がある

AI は便利だが、間違ったコードを書くこともある。そのとき、「AI が変更する前の状態に戻す」ことが必要になる。Git があれば、これがワンクリックでできる。

### 理由 3: 成果物を公開・共有する手段が GitHub

AI に作ってもらったウェブサイトやツールを公開するには、GitHub にアップロードするのが最も簡単である。GitHub Pages を使えば、無料でウェブサイトを公開できる。

## AI ツール別「Git をどこまで使うか」早見表

AI コーディングツールによって、Git との関わり方が異なる。自分が使いたいツールを確認しておこう。

| ツール             | Git の必要度 | Git がないとどうなるか                                                    |
| ------------------ | ------------ | ------------------------------------------------------------------------- |
| **Claude Code**        | 必須         | インストール時にエラー。プロジェクトフォルダに `.git` がないと動作しない      |
| **Codex**              | 必須         | リポジトリ単位でタスクを実行するため、Git なしでは使えない                    |
| **Google Antigravity** | 必須         | Git リポジトリ内で動作。`.git` がないとコンテキスト理解が制限される           |
| **GitHub Copilot**     | ほぼ必須     | VS Code上で動くが、Git リポジトリがないとコンテキスト理解が大幅に劣化する     |
| **Cursor**             | 推奨         | Git なしでも動くが、変更の追跡ができないため実用的でない                      |

どのツールを使う場合でも、 **Git を知っていることは AI コーディングの前提条件** と考えてよい。

## 本書のアプローチ — 「Git を学ぶ」のではなく「通過する」

一般的な Git 入門書は、Git の内部構造や設計思想から丁寧に解説する。それはエンジニアにとっては有益だが、「AI ツールを使いたいだけ」の人には遠回りである。

本書のアプローチはこうだ。

> **Git は目的ではなく手段。AI コーディングの入口として、必要最小限の Git を最短距離で「通過」する。**

具体的には、以下の方針を取る。

1. **画面操作（GUI）が先**。黒い画面（CLI）は「裏側の解説」として扱う
2. **AI ツールの実演が先**。まず「こうなる」を見せてから、仕組みを説明する
3. **完璧を求めない**。「動く」がゴール。深い理解は後から付いてくる

## この本を読み終えたらできること

最終章（第 8 章）まで読み終えると、こんなことができるようになる。

- Claude Code に「こういうアプリを作って」と頼んで、実際にアプリが動く
- AI が書いたコードの変更内容を確認できる
- 気に入らなければ、AI の変更を元に戻せる
- 完成したアプリを GitHub に公開して、誰でもアクセスできるようにする

**Git を「使いこなす」必要はない。AI と協働するために「通過できる」ようになれば、それで十分である。**
