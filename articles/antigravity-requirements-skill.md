## はじめに

AIで社内ツールを作ろうとして、スタートで詰まった。

---

「経費精算を楽にしたい」と伝えた。

AIは「具体的にどのような機能が必要ですか？」と聞いてきた。

...わからない。だから聞いてるのに。

---

「レシート撮影と自動入力」と答えた。

AIはスマホアプリを作り始めた。

違う違う、そうじゃない。GASでいいのに...

---

非エンジニアの個人開発で一番つらいのは、 **「何をどう伝えればいいかわからない」** こと。

本記事では、 **AIの方から質問してもらって要件を整理し、フォルダ名まで提案してもらうスキル** の詳細を解説する。

## 前提条件

- Google Antigravity インストール済み
  - まだの場合 → [Antigravityインストールガイド](antigravity-windows-install-guide)
- プログラミング経験は不要

## 解決策：要件定義スキル

### コンセプト

**非エンジニアが複雑な指示を書かなくても、AIが質問しながら必要な情報を引き出す。フォルダ名も提案してくれる。**

```
従来:
ユーザー: （フォルダ名で30分悩む）
ユーザー: 詳細な指示を書く（でも何を書けばいいかわからない）
    ↓
AI: 「具体的には？」（また聞かれる...）

新しいフロー:
ユーザー: 「経費精算を楽にしたい」（これだけ）
    ↓
AI: 「誰が使いますか？」（質問してくれる！）
    ↓
（5つの質問に答えるだけ）
    ↓
AI: フォルダ名は `team-expense-assistant` でいかがですか？
    （理由: チーム利用であること、業務を補助する役割を明確にするため）
    ↓
ユーザー: はい
    ↓
AI: ✅ フォルダを作成しました
```

### 質問フロー

AIが聞くのは、Q0 + 5つの質問。

| # | 質問 | なぜ聞くのか |
|---|------|-------------|
| 0 | 今回はどんなものを作りたいですか？ | プロジェクトの全体像を把握 |
| 1 | 誰が使いますか？ | 自分だけ？チーム？で作り方が変わる |
| 2 | 必須の機能を3つ挙げてください | あれもこれも、を防ぐ |
| 3 | いつまでに必要ですか？ | 急ぎなら簡易版から |
| 4 | 外部サービスの費用は出せますか？ | 無料で作るか、有料API使うか |
| 5 | 最終的に誰に見せますか？ | 上司向け？自分用？で仕上がりが変わる |

**この質問に答えるだけで、AIが要件を整理し、センスのいいフォルダ名まで提案してくれる。**

## 実装方法

### Antigravityに指示を出す

`skills/requirements-gathering` フォルダで以下を実行:

```
skills/requirements-gathering/SKILL.md を作成してください。
フォルダがなければ作成してください。

【解決したい悩み】
非エンジニアの個人開発で、
1. フォルダ名で30分悩む（センスのいい名前にしたい）
2. 何を伝えればいいかわからない
3. 伝えたつもりが違うものができる

【トリガー】
- 「〇〇を作りたい」
- 「〇〇が欲しい」
- 「〇〇を自動化したい」
- 「〇〇を楽にしたい」

【質問フロー（この順序を必ず守ること）】
Q0: 今回はどんなものを作りたいですか？（必須・最初に聞く）
Q1: 誰が使いますか？
Q2: 必須の機能を3つ挙げてください
Q3: いつまでに必要ですか？
Q4: 外部サービスの費用は出せますか？
Q5: 最終的に誰に見せますか？

【出力】
- 要件サマリー（プロジェクト概要、要件詳細、技術方針）
- 推奨フォルダ名（半角英数字・ハイフンのみ）+ 命名理由
- 承認後にフォルダを作成

【フォルダ名の命名ルール】
- 半角英数字とハイフンのみ
- プロジェクトの目的が一目でわかる名前
- 命名理由も一緒に提示する

【制約事項】
- 質問の省略禁止: Q0〜Q5 はすべて実行すること
- 順序変更禁止: 必ず Q0 → Q1 → Q2 → Q3 → Q4 → Q5 の順で実行
- 1問ずつ対話: 複数の質問を一度に聞かない
- 「わからない」「未定」も有効な回答として受け入れる
- 非エンジニアにもわかる言葉で説明する
```

### 作成されるファイル構成

```
Projects/
├── skills/
│   └── requirements-gathering/
│       └── SKILL.md              ← 要件定義スキル
└── team-expense-assistant/       ← 自動作成される（センスのいい名前！）
```

## デモ：実際の対話フロー

![5つの質問で深掘り](/images/requirements-questions.png)
*AIが1問ずつ質問し、回答を確認しながら進む。*

```
経費精算を楽にするツールを作りたい
```

**Q1: ユーザー**

```
AI: 経費精算ツールの開発ですね！いくつか質問させてください。
    このツールは誰が使いますか？
    例: 自分だけ、チームの5人、部署全員

ユーザー: 自分とチームの5人
```

**Q2: 機能**

```
AI: 絶対に必要な機能を3つ挙げてください。
    （3つに絞ることで、最初に作るものを明確にします）

ユーザー: レシート撮影、自動入力、月次レポート
```

**Q3: 期限**

```
AI: いつまでに必要ですか？
    例: 今週中、来月末、急ぎではない

ユーザー: 来月末まで
```

**Q4: 予算**

```
AI: 外部APIやサービスの費用は出せますか？
    例: 無料のみ、月1000円まで、必要なら相談

ユーザー: 無料でお願いしたい
```

**Q5: 成果物**

```
AI: 最終的に誰に見せますか？
    例: 自分用、上司に承認してもらう、お客様に提出

ユーザー: 経理部長に承認してもらう
```

**要件サマリー + フォルダ名提案**

![要件サマリーとフォルダ名提案](/images/requirements-summary.png)
*回答が構造化され、フォルダ名も自動で提案される。*

```
AI: では、要件をまとめます。

## 要件サマリー

### プロジェクト概要
チーム向け経費精算効率化ツール

### 要件詳細
| 項目 | 内容 |
|------|------|
| ユーザー | 自分とチーム5人 |
| 必須機能 | レシート撮影、自動入力、月次レポート |
| 期限 | 来月末 |
| 予算 | 無料のみ |
| 成果物 | 経理部長への報告用 |

### 技術的な方針（提案）
- Google Apps Script + Spreadsheet（無料で使える）
- Gemini API（無料枠）でOCR

---

📁 フォルダ名は `team-expense-assistant` でいかがですか？

**命名理由**: 
- `team-` → チームで使うツールであることが明確
- `expense-` → 経費関連であることがわかる
- `-assistant` → 業務を補助する役割を示す

この内容で開発を進めてよろしいですか？

ユーザー: はい

AI: ✅ フォルダ「team-expense-assistant」を作成しました
```

**30分悩んでいたフォルダ名が、5つの質問に答えるだけで決まった。しかもセンスがいい。**

## フォルダ名の提案例

| やりたいこと | 提案されるフォルダ名 | 命名理由 |
|--------------|---------------------|----------|
| 経費精算を楽にしたい | `team-expense-assistant` | チーム利用 + 経費 + 補助 |
| 日報を自動化したい | `daily-report-bot` | 日次 + レポート + 自動化 |
| FAQを作りたい | `faq-system` | FAQ + システム |
| 議事録を自動作成したい | `meeting-notes-ai` | 会議 + メモ + AI活用 |
| 請求書を自動で作りたい | `invoice-generator` | 請求書 + 生成 |

**もうフォルダ名で悩まない。命名センスはAIに任せる。**

## 非エンジニアの個人開発でありがちな失敗

| 失敗パターン | 原因 | このスキルでの解決 |
|-------------|------|-------------------|
| フォルダ名で30分悩む | 命名センスがない | AIが理由付きで提案 |
| 「レシート撮影」→スマホアプリ | 技術制約を伝えてない | Q4で「無料」→GAS選択 |
| 「自動入力」→OCR精度の話 | やりたいことが曖昧 | Q2で「3つに絞る」→明確化 |
| 3日経っても完成しない | スコープが広がりすぎ | Q2で「3つだけ」→最小限 |

## つまずきポイント

### 1. スキルファイルが作成されない

**問題**: Antigravityに「スキルを作成して」と指示しても、フォルダが作成されないことがある。

**解決策**: 「フォルダがなければ作成して」と明示的に指示する。

### 2. 質問が一度に全部表示される

**問題**: 5つの質問がまとめて表示されてしまう。

**解決策**: 「1問ずつ対話形式で」と指示に明記する。

### 3. フォルダ名が気に入らない

**問題**: AIが提案したフォルダ名がしっくりこない。

**解決策**: 「別の候補も出して」と言えば、複数の候補を提案してくれる。

## まとめ

- **非エンジニアでも要件定義できる** — AIが質問してくれるから
- 5つの質問に答えるだけで要件が整理される
- **フォルダ名もAIが提案** — 命名センス不要、理由付きで安心
- 「無料」「来月末」などの制約が、技術選定に反映される
