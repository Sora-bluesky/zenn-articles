## はじめに

2月のAnthropicからの請求額：**$323.63**（約48,500円）。

![2月のAnthropicからのReceipt一覧](/images/anthropic-receipts-february.png)

Max 20xプランで使っていたが、それでもリミットにかかった。設定画面で「追加使用量」を有効にし、残高をチャージして使い続けた結果がこれだ。記事の検証やClaude Codeの実験を繰り返した結果とはいえ、初めて見た時は正直青ざめた。

![追加使用量の設定画面（$321.02使用、残高$89.85）](/images/claude-additional-usage-settings.png)

Anthropic公式の統計によると、開発者1人あたりの**1日の平均コストは約$6**（約900円）。90%のユーザーは1日$12以下に収まるという。つまり月額の目安は**$100〜$200**（約15,000〜30,000円）。

$323は「使いすぎ」の部類に入る。だからこそ、コスト管理の方法を真剣に調べた。この記事では、公式ドキュメントに基づいたコスト管理術をまとめる。

:::message
**この記事の前提**
- 筆者は **Max 20x プラン**（月額 $200）を利用。リミット到達後は「追加使用量」を有効にしてチャージして使い続けた結果が $323.63
- Max プランでもリミット超過時は追加料金が発生するため、**どのプランでもコスト管理は必要**
:::

## コストの仕組み

### 何にお金がかかっているのか

Claude Codeの料金は**トークン消費量**で決まる。トークンとは、AIが処理するテキストの単位。

コストに影響する要因：

| 要因 | 影響 |
|------|------|
| 読み込むファイルの数 | ファイルを読むたびにトークン消費 |
| 会話履歴の長さ | 長い会話ほど毎回のリクエストでトークンが累積 |
| クエリの複雑さ | 複雑な質問ほど推論にトークンを使う |
| コンパクトの頻度 | 定期的にコンパクトすれば消費を抑制可能 |

### バックグラウンドでも消費する

意外と知られていないが、Claude Codeは**何もしていなくてもトークンを消費する**。

| バックグラウンド処理 | 内容 |
|-------------------|------|
| 会話要約 | `--resume` 機能のために、前の会話を要約するジョブが自動で走る |

ただし、その消費量はセッションあたり**$0.04未満**（約6円）と微々たるもの。気にする必要はない。

## `/cost` で現状を把握する

まずは自分がどれくらい使っているかを知ることが大事。

### 使い方

Claude Codeのセッション中に入力する：

```
/cost
```

### 表示される情報

| 項目 | 内容 |
|------|------|
| Total cost | 現在のセッションの合計コスト |
| Total duration (API) | API処理にかかった時間 |
| Total duration (wall) | 実際の経過時間 |
| Total code changes | コードの追加・削除行数 |

:::message alert
**Max/Proプランでは使えない**
`/cost` コマンドはAPI従量課金のユーザーのみ対応。定額制プランでは表示されない。
:::

### 過去の使用状況を確認する

Anthropic Console（https://console.anthropic.com）にログインすると、過去の使用状況を確認できる。ワークスペースの支出制限も設定可能。

## コスト削減の6つのテクニック

### 1. `/compact` で会話を圧縮する

最も効果的なコスト削減方法。会話が長くなるほど、毎回のリクエストで送信されるトークンが増える。`/compact` で会話を要約し、トークン消費を抑える。

```
/compact
```

**自動コンパクト**もある。コンテキストが容量の**95%**を超えると自動的に発動する。

**カスタム指示つき**で、保持したい情報を指定することもできる：

```
/compact テスト結果と変更したファイルの一覧は必ず残して
```

### 2. `/clear` でコンテキストをリセットする

関連のないタスクに移る時は、会話を続けずにリセットする。

```
/clear
```

「さっきのファイルを読んだ情報を引き継ぎたい」と思うかもしれないが、不要な情報がコンテキストに残る方がコストもパフォーマンスも悪化する。

:::message
**2回修正しても直らなかったら `/clear`**

失敗した試行錯誤がコンテキストに残り、さらに混乱する。学んだことを活かして、最初からやり直す方が速く、安い。
:::

### 3. 具体的な指示を出す

曖昧な指示は、不要なファイルスキャンをトリガーしてトークンを浪費する。

```
❌ 「このバグを直して」
✅ 「src/auth/session.ts の refreshToken 関数でエラーが出る。修正して」
```

ファイルパスを明示するだけで、Claudeが探索に費やすトークンを大幅に節約できる。

### 4. サブエージェントに調査を委譲する

大量のファイルを読む「調査」はコストが高い。サブエージェントは別のコンテキストで動くため、メインの会話のトークン消費を抑えられる。

```
サブエージェントを使って、認証システムの仕組みを調べて
```

詳しくは[第5部：サブエージェント活用術](claude-code-subagents)を参照。

### 5. Plan Mode で無駄な実装を防ぐ

いきなり実装させると、間違った方向に進んでやり直しが発生する。Plan Mode で先に計画を立てれば、無駄なコードの生成を防げる。

```
Plan Mode に切り替えて。
まず src/auth/ を読んで、セッション管理の仕組みを理解して。
```

詳しくは[第4部：プロンプト術](claude-code-effective-prompts)を参照。

### 6. CLAUDE.md を膨らませすぎない

CLAUDE.md はすべてのリクエストで読み込まれる。つまり、1行増えるたびに**全リクエストのコストが上がる**。

**各行について自問する**：「これを削除したら、Claudeが間違いを犯すか？」

答えが No なら、その行は不要。

## Max Plan vs API：どちらを選ぶか

| | Claude Max 5x | Claude Max 20x | API（従量課金） |
|---|-------------|---------------|---------------|
| 料金体系 | 月額 $100 | 月額 $200 | 使った分だけ |
| 利用上限 | 基本の5倍 | 基本の20倍 | 支出制限を自分で設定 |
| 上限到達時 | 一時停止 or 追加使用量 | 一時停止 or 追加使用量 | 制限に達したら停止 |
| コスト管理 | 上限超過に注意 | 上限超過に注意 | `/cost` で監視 |
| 向いている人 | 日常的に使う人 | ヘビーユーザー | たまに使う人、チーム利用 |

:::message alert
**Max プランでも油断禁物**
筆者は Max 20x（$200/月）を利用していたが、記事の検証で使い込んだ結果、リミットに到達。「追加使用量」を有効にしてチャージして使い続けた結果、合計 **$323.63** になった。「定額だから安心」とは限らない。
:::

**判断基準**：
- 毎日数時間使うなら → **Max Plan**がベース（ただし上限超過に注意）
- 週に数回、短時間なら → **API**の方が安くなる可能性
- チームで使うなら → **API**（ワークスペース管理、支出制限が使える）

## コストを下げる使い方の習慣

| 習慣 | 効果 |
|------|------|
| タスクが変わったら `/clear` | 不要なコンテキストの累積を防ぐ |
| 長い会話の途中で `/compact` | トークン消費の増加を抑える |
| ファイルパスを明示する | 探索コストの削減 |
| 調査は「サブエージェント」 | メインコンテキストの保護 |
| CLAUDE.md は短く | 全リクエストの基礎コストを下げる |
| 2回失敗したら `/clear` | 失敗のコンテキスト蓄積を防ぐ |

## まとめ

| ポイント | 内容 |
|---------|------|
| 平均コスト | 1日約$6（月$100〜$200） |
| 最重要テクニック | `/compact` と `/clear` |
| 根本原則 | コンテキスト管理 = コスト管理 |
| 計測方法 | `/cost` コマンド |
| プラン選択 | Max でも上限超過に注意。コスト管理はどのプランでも必要 |

**覚えておくこと**：

> コンテキストの管理はパフォーマンスだけでなく、コストにも直結する。
> `/clear` と `/compact` は最も費用対効果の高いコマンド。
