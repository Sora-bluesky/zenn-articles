## はじめに

Claude Codeで開発していて、こんな経験はないだろうか。

- 昨日の続きをやろうとしたら、Claude Codeが「初めまして」状態になっていた
- 「あの時のセッション、どこだっけ？」と過去の作業を探したい
- 複数の機能を行き来しているうちに、どのセッションでどの作業をしていたか分からなくなった

Claude Codeの会話は **自動的にローカルに保存されている**。再開する方法を知っていれば、作業を途切れさせずに済む。

この記事では、公式ドキュメントに基づいてセッション管理の全体像を解説する。

## セッションの仕組み

まず、Claude Codeのセッションがどう管理されているかを知っておこう。

- **自動保存**：すべての会話は、完全なメッセージ履歴を含めてローカルに自動保存される
- **プロジェクト単位**：セッションはプロジェクトディレクトリ（フォルダ）ごとに保存される
- **Git連携**：`/resume` ピッカーは、[Git Worktree](claude-code-parallel-worktrees)を含む同じGitリポジトリからのセッションも表示する

つまり、Claude Codeを閉じてもセッションは消えない。翌日でも翌週でも、いつでも再開できる。最初にこれを知ったとき「え、保存ボタンどこ？」と探し回ったのは内緒だ。

再開時には以下が復元される：

| 復元されるもの | 説明 |
|--------------|------|
| メッセージ履歴 | やり取りの全内容 |
| ツール状態 | 前回のファイル編集やコマンド実行の結果 |
| コンテキスト | 以前のコンテキストがすべて保持される |
| モデルと設定 | 元のセッションと同じモデル・設定で再開 |

## 再開の3つの方法

### 方法1：`--continue` で直近のセッションを続行

最もシンプルな方法。今いるディレクトリで、最後に使ったセッションをそのまま続ける。

```bash
claude --continue
```

**使いどころ**：
- 昨日の続きをやりたい
- Claude Codeを一度閉じて再起動した後

### 方法2：`--resume` でセッションを選んで再開

過去のセッション一覧から選んで再開する。

```bash
# セッションピッカーを開く（一覧から選ぶ）
claude --resume

# セッション名を指定して直接再開
claude --resume auth-refactor
```

**使いどころ**：
- 複数のセッションから特定の作業を探したい
- 名前をつけたセッションに戻りたい

セッション中に別のセッションに切り替えたい場合は、`/resume` コマンドも使える。

```
> /resume
```

### 方法3：`--from-pr` でPRに紐づいたセッションを再開

`gh pr create` でPR（プルリクエスト）を作成すると、そのセッションは自動的にPRにリンクされる。PR番号を指定して再開できる。

```bash
claude --from-pr 123
```

**使いどころ**：
- PRのレビューコメントに対応したい
- PRに関連する作業の続きをしたい

## セッションに名前をつける

セッション管理で **最も効果的なのが名前をつけること** だ。

名前がないと、セッションは最初のプロンプト（「explain this function」など）で表示される。後から探すのが困難になる。

### 名前をつける

セッション中に `/rename` コマンドを使う。

```
> /rename auth-refactor
```

### 名前で再開する

```bash
# コマンドラインから
claude --resume auth-refactor

# セッション中から
> /resume auth-refactor
```

### セッションピッカーから名前を変更する

`/resume` でピッカーを開き、対象セッションに移動して `R` を押すと、その場で名前を変更できる。

:::message
**名前付けのコツ**

作業内容が一目でわかる名前をつける。

- `auth-refactor` → 認証リファクタリング
- `bugfix-login` → ログインバグ修正
- `feature-notification` → 通知機能の開発

「explain this function」よりも「payment-integration」の方が、後で見つけやすい。
:::

## セッションピッカーの使い方

`/resume`（または `claude --resume`）で開くセッションピッカーには、便利なショートカットがある。

### 基本操作

| ショートカット | アクション |
|:-------------|:----------|
| `↑` / `↓` | セッション間を移動 |
| `Enter` | 選択したセッションを再開 |
| `P` | セッションの中身をプレビュー |
| `R` | セッションの名前を変更 |
| `Esc` | ピッカーを閉じる |

### フィルタリング

| ショートカット | アクション |
|:-------------|:----------|
| `/` | 検索してセッションを絞り込む |
| `A` | 現在のディレクトリ / 全プロジェクトを切り替え |
| `B` | 現在のGitブランチのセッションだけ表示 |

### グループ化されたセッション

| ショートカット | アクション |
|:-------------|:----------|
| `→` / `←` | グループを展開 / 折りたたむ |

`/rewind` や `--fork-session` で作成したフォークセッションは、元のセッションの下にグループ化される。`/rewind` の詳細は次のセクションで解説する。

### 表示される情報

ピッカーの各セッションには以下が表示される：

- セッション名（または最初のプロンプト）
- 最後のアクティビティからの経過時間
- メッセージ数
- Gitブランチ（該当する場合）

## `/rewind` で過去の地点に巻き戻す

セッションを「巻き戻す」機能が `/rewind` だ。過去の任意のメッセージまで遡り、そこからやり直すことができる。

### 使い方

セッション中に入力する：

```
/rewind
```

メッセージの一覧が表示されるので、戻りたい地点を選択する。

### 復元オプション

巻き戻し地点を選択すると、以下の5つのオプションが表示される。

| オプション | 動作 |
|:---------|:-----|
| **Restore code and conversation** | コードと会話の両方を選択地点の状態に復元する |
| **Restore conversation** | 会話のみを選択地点に戻す。コードの変更はそのまま残る |
| **Restore code** | コードのみを選択地点に戻す。会話はそのまま残る |
| **Summarize from here** | 選択地点以降の会話を要約に圧縮する。コンテキストの節約に有効 |
| **Never mind** | 何もせずに戻る |

:::message
**Summarize from here の活用**
会話が長くなりすぎた場合、途中の地点を選んで「Summarize from here」を実行すると、選択地点以降のやり取りが要約に圧縮される。`/compact` がセッション全体を要約するのに対し、こちらは特定の地点以降だけを対象にできる。コンテキストを節約しつつ、重要な前半部分はそのまま残したい場合に便利だ。
:::

`/rewind` で作成されたセッションはフォークとして元のセッションの下にグループ化される。元のセッションが失われることはない。

## 実践パターン

### パターン1：タスク別にセッションを管理する

複数のタスクを抱えている場合、タスクごとにセッションを分けて名前をつけておく。

```
# タスクAの作業を開始
> /rename feature-user-profile

# 別のタスクに切り替えたい
> /resume
# → ピッカーからタスクBのセッションを選ぶ

# タスクAに戻りたい
> /resume feature-user-profile
```

### パターン2：Git Worktreeと組み合わせる

[第6部](claude-code-parallel-worktrees)で解説したGit Worktreeと組み合わせると、セッション管理がさらに強力になる。

```bash
# Worktree 1で機能開発
cd ../my-project-feature-a
claude --continue  # 前回のセッションを続行

# Worktree 2でバグ修正（別ターミナル）
cd ../my-project-bugfix
claude --continue  # こちらも前回のセッションを続行
```

Worktreeごとにセッションが独立しているため、それぞれの作業が混ざらない。

### パターン3：ヘッドレスモードで自動化する

スクリプトからセッションを再開して、非インタラクティブに実行することもできる。

```bash
claude --continue --print "テストを実行して結果を報告して"
```

`--print`（短縮形 `-p`）をつけると、対話なしで結果だけ出力される。CI/CDや定期タスクに使える。

## まとめ

| やりたいこと | コマンド |
|-------------|---------|
| 直近のセッションを続行 | `claude --continue` |
| セッション一覧から選ぶ | `claude --resume` |
| 名前を指定して再開 | `claude --resume <名前>` |
| PRに紐づいたセッションを再開 | `claude --from-pr <番号>` |
| セッションに名前をつける | `/rename <名前>` |
| セッション中に別のセッションへ切り替え | `/resume` |

**ベストプラクティス**：

- 作業を始めたら、すぐに `/rename` で名前をつける
- `--continue` を日常的に使い、昨日の続きをシームレスに再開する
- 複数タスクを抱えている場合は、タスクごとにセッションを分けて名前で管理する

セッション管理を習慣にすれば、「あの作業どこまでやったっけ？」がなくなる。
