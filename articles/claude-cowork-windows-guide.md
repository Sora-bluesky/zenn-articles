## この記事で分かること

- **Cowork とは何か** — フォルダを選んで指示するだけで、AIが自律的にファイル操作してくれる機能
- **Claude Code との違い** — 中身は同じ技術。ターミナル不要のGUIで誰でも使える
- **Windows版セットアップ** — ダウンロードからタスク実行まで4ステップ

## 経緯

2026年2月10日、Claude公式アカウントがポストした。

> Cowork is now available on Windows.

ついに来た。

Claude Cowork は、1月にmacOS限定でリリースされた「AIエージェント」機能。フォルダを指定して「整理して」と言えば、Claude が自律的にファイルの分類・リネーム・作成をやってくれる。Claude Code のエージェント機能を、ターミナルなしで誰でも使えるようにしたもの——と言えばいいだろうか。

macOS限定だったので「Windowsは蚊帳の外か」と思っていたが、約1ヶ月でmacOS版と同じ機能がすべて使えるWindows版が出た。しかも、グローバル/フォルダインストラクションという新機能つきで。

## Cowork とは

### ひとことで言うと

**「フォルダを指定して、やりたいことを伝えれば、あとはClaudeが勝手にやってくれる」機能。**

チャットでは「質問 → 回答」の繰り返し。Coworkでは「タスクを渡す → Claudeが計画を立てて自律的に作業 → 完了報告」という流れになる。

### Claude Code との違い

ここが一番聞かれるところだと思う。

| 項目 | Claude Code | Cowork |
|------|-------------|--------|
| 操作方法 | ターミナル（黒い画面） | デスクトップアプリのGUI |
| 対象ユーザー | 開発者向け | **誰でも** |
| セットアップ | コマンド入力が必要 | **フォルダを選ぶだけ** |
| 得意なこと | コーディング、Git操作 | ファイル整理、資料作成、データ抽出 |
| サンドボックス（操作範囲を制限する安全機能） | 設定が必要 | **最初から有効** |
| OS | Windows / macOS / Linux | **Windows / macOS** |

つまり、**中身は同じ技術**。包装が違う。

Claude Code を使っていた人にとっては「ターミナルなしで、同じことがGUIからできるようになった」という理解でいい。Claude Code を使ったことがない人にとっては「AIに雑務を任せられる、初めてのエージェント体験」になる。

### 必要なもの

| 項目 | 要件 |
|------|------|
| OS | Windows 10 以降 |
| 契約 | Claude Pro（$20/月）以上 |
| アプリ | Claude デスクトップアプリ最新版 |
| ネット | 常時接続が必要 |

:::message
**Pro プランでも使える**
現在は Pro（月額$20 / 約3,000円）でも利用可能。ただし、Pro は使用量制限に達しやすい。Cowork のタスクは通常のチャットより多くのトークンを消費するため、頻繁に使うなら Max（月額$100〜 / 約15,000円〜）が安心。
:::

## Windows版セットアップ

### Step 1：Claude デスクトップアプリをダウンロード

[claude.com/download](https://claude.com/download) にアクセスし、Windows版をダウンロードしてインストール。

既にインストール済みの場合は、最新版にアップデートする。

### Step 2：Cowork タブに切り替える

アプリを起動すると、上部に「**Chat**」「**Cowork**」のモード切替がある（Claude Code をインストール済みの場合は「Code」タブも表示される）。

「**Cowork**」をクリック。

### Step 3：フォルダを選択

入力欄の下に「**Work in a folder**」というチェックボックスがある。これにチェックを入れて、Claude にアクセスを許可するフォルダを選択する。

:::message alert
**フォルダ選択の注意点**
- Claude はそのフォルダ内のファイルを読み書き・削除できる
- 最初は**テスト用フォルダ**で試すこと
- 重要なファイルが入ったフォルダは避ける
- バックアップを取っておく
:::

### Step 4：タスクを入力

あとは「何をしてほしいか」を入力するだけ。

```
このフォルダのファイルを種類別に整理して
```

Claude が計画を立てて、確認を求めてから作業を始める。

## 新機能：グローバル / フォルダインストラクション

Windows版リリースと同時に追加された新機能。Claude Code の設定ファイル（`CLAUDE.md`）に相当するものだと思えばいい。

### グローバルインストラクション

**すべてのセッションに適用される指示。**

例えば「日本語で応答して」「です・ます調で書いて」「ファイルの削除は絶対にしないで」など、自分の好みや安全ルールを設定しておける。

Claude Code では設定ファイルを手動で編集する必要があったが、Coworkでは**チャットから直接設定・更新できる**ようになった。

```
グローバルインストラクションに以下を追加して：
- 日本語で応答すること
- ファイルを削除する前に必ず確認すること
- 作成するファイルはUTF-8エンコーディングにすること
```

### フォルダインストラクション

**特定のフォルダで作業する時だけ適用される指示。**

例えば「経費フォルダ」を選んだ時だけ「金額は税込みで記載して」「ファイル名は YYYY-MM-DD_内容.xlsx にして」といったルールを適用できる。

Claude Code では `.claude/CLAUDE.md`（プロジェクト単位）で設定していたものに近い。

| 設定の種類 | Claude Code | Cowork |
|-----------|-------------|--------|
| 全体に適用 | 設定ファイルを手動編集 | グローバルインストラクション |
| フォルダ単位 | 設定ファイルを手動編集 | フォルダインストラクション |
| 設定方法 | テキストエディタで直接書く | **チャットで指示するだけ** |

Claude Code 使いとしては、この「チャットから設定できる」のが地味にうれしい。

## やってみた①：ダウンロードフォルダの整理

Cowork 体験記事の定番ネタ。macOS版のレビュー記事では、どこもこれを最初に試していた。Windows でも同じことができるか確認する。

### Claude Code でやる場合（従来）

ターミナル（黒い画面）を開いて、フォルダ移動コマンドを打って、`claude` を起動して……という手順が必要だった。

### Cowork でやる場合

1. Coworkタブを開く
2. 「Work in a folder」でダウンロードフォルダを選択
3. 入力欄に書く：

```
このフォルダのファイルを種類別にサブフォルダに分類して。
ファイル名は YYYY-MM-DD_元のファイル名 にリネームして。
削除はしないで。
```

これだけ。ターミナルを開く必要がない。`cd`（フォルダ移動コマンド）もいらない。

:::message
**Claude Code 使いの所感**
やっていることは同じ。だけど「ターミナルを開かなくていい」という一点で、心理的なハードルが全然違う。自分はClaude Codeに慣れたけど、会社の同僚に「ターミナル開いて」と言うのは気が引ける。Coworkなら「このアプリ開いて、フォルダ選んで、これ入力して」で伝わる。
:::

## やってみた②：散らばったメモから報告書を作成

非エンジニアの実務に近いユースケース。

### シナリオ

デスクトップに散らばっているテキストファイル、Wordファイル、メモ書き。これらを一つの報告書にまとめたい。

### やり方

1. 報告書の元ネタが入ったフォルダを選択
2. 指示を入力：

```
このフォルダ内のファイルをすべて読み込んで、内容をまとめた報告書を作成して。

報告書の形式：
- Word形式（.docx）で出力
- 冒頭に概要（200字程度）
- 内容ごとにセクション分け
- 最後に「次のアクション」を3つ提案

ファイル名：報告書_2026-02-11.docx
```

Claude Code でも同じことはできるが、Coworkの方が「計画→確認→実行」のステップが視覚的に見えるので、非エンジニアにとっては安心感がある。

## Claude Code との使い分け

約1ヶ月、macOS版のCoworkを横目で見ながらClaude Codeを使い続けてきた。Windows版が出て実際に触った感触を踏まえて、使い分けの目安をまとめる。

| やりたいこと | おすすめ |
|-------------|---------|
| コーディング | Claude Code |
| Git 操作（コミット、PR） | Claude Code |
| ファイル整理 | **Cowork** |
| 資料作成（Word、Excel、PDF） | **Cowork** |
| データ抽出（レシート→Excel等） | **Cowork** |
| サブエージェント＋スキル活用 | Claude Code（柔軟性が高い） |
| 同僚に教える | **Cowork**（ターミナル不要） |

一言でまとめると：

- **作る人** → Claude Code
- **使う人** → Cowork

両方使える環境があるなら、目的に応じて切り替えるのがいい。

## 注意点

### 1. レート制限（使用量上限）

Cowork は通常のチャットより多くのトークンを消費する。Pro プランだとすぐ上限に達する可能性がある。

上限に達すると一時的に使えなくなる。使用量は数時間でリセットされるので（2026年2月時点）、時間を置いてから再開すればいい。

### 2. アプリを閉じるとセッション終了

Cowork での作業中は、Claude デスクトップアプリを開いたままにしておく必要がある。アプリを閉じると、そのセッションは終了する。

### 3. セキュリティ

- Claude は指定フォルダ内のファイルを**削除する能力**がある。明確な指示を出すか、「削除しないで」と明記する
- **不正な指示の埋め込み**（プロンプトインジェクション）のリスクがある。インターネットから取得したファイルに、Claude の動作を変えようとする隠し指示が含まれている可能性がある
- 機密情報（パスワード、認証情報）が入ったフォルダは指定しない

:::message
**心配しすぎなくても大丈夫**: テスト用フォルダで試せば安全。重要なファイルさえ避ければ、気軽に使える。
:::

### 4. リサーチプレビュー

まだ正式リリースではない。不安定な動作やバグがある可能性がある。重要な業務に使う場合は、必ず結果を自分の目で確認すること。

## プラグイン

Cowork には公式のプラグインが11種類用意されている（2026年2月時点）。プラグインをインストールすると、特定の業務フローに特化した動作をさせられる。

| カテゴリ | 内容 |
|---------|------|
| Productivity | タスク管理、カレンダー、日常ワークフロー |
| Enterprise search | 社内ツール横断検索 |
| Sales | 見込客リサーチ、商談準備 |
| Finance | 財務分析、モデル構築 |
| Plugin Create | **カスタムプラグインを自分で作る** |

プラグインは公式サイトで公開されており、自作も可能。Claude Code のスキル機能に近い概念だ。

:::message
**プラグインのインストール方法**
Cowork の左サイドバーにある「Plugins」から、利用可能なプラグインを閲覧してインストールできる。カスタムプラグインのアップロードも可能。
:::

| ポイント | 内容 |

| 何が新しい | グローバル / フォルダインストラクション（チャットから設定可能） |
| 誰向け | Windows ユーザー、非エンジニア、Claude Pro/Max 契約者 |

| 注意点 | レート制限、アプリ閉じると終了、セキュリティに注意 |

Claude Code は「開発者のためのエージェント」だった。Cowork は「すべての人のためのエージェント」。そして、ようやく Windows ユーザーも仲間に入れた。

ターミナルを開く必要がない。`cd` も `claude` も打たなくていい。フォルダを選んで、やりたいことを書くだけ。

非エンジニアの自分にとって、Claude Code のハードルは「ターミナル」だった。Cowork はそのハードルを取り払ってくれた。

## 次のステップ

1. **まず試す**: [Claude デスクトップアプリ](https://claude.com/download)をダウンロードして、Coworkタブを開く
2. **テストフォルダで練習**: デスクトップに「テスト」フォルダを作り、適当なファイルを入れて「整理して」と指示してみる
3. **インストラクションを設定**: 慣れたら「日本語で応答して」「削除は確認してから」をグローバルインストラクションに追加
