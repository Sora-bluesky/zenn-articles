## はじめに

AIにコードを書かせた後、こんな不安を感じたことはないだろうか。

- 「本当にちゃんと直ったの？」
- 「何が変わったのかよく分からない」
- 「AIが勝手に変な変更をしていないか心配」

Google Antigravity の **Artifacts（アーティファクト）** は、この不安を解消する機能。
AIが何をしたかを「スクリーンショット」「動画」「タスクリスト」など目に見える形で提示してくれる。

この記事では、Artifactsの仕組みと使い方を非エンジニア向けに解説する。

---

## Artifacts とは？

**Artifacts = AIの作業報告書**

AIが「バグを修正しました」と言ったとき、本当に直ったかどうかをどう確認する？

これまでのAIツールでは：

- コードの変更箇所を自分で読む（読めない人には無理）
- AIの「直しました」という言葉を信じるしかない

Antigravity の Artifacts は、AIが自分の作業を **証拠付きで報告** する仕組み。

:::message
**例え話**
リフォーム業者に「壁を直しました」と言われたとき、ビフォーアフターの写真を見せてくれるのがArtifacts。写真なしで「直しました」とだけ言われるのが従来のAIツール。
:::

## Artifacts の主な種類

Artifacts には複数の種類がある（コード差分やアーキテクチャ図なども生成されることがある）。ここでは非エンジニアに特に役立つ5つを紹介する。

### 1. タスクリスト --- 「これからやること」の一覧

AIがコードを書き始める前に、「こういう手順で進めます」という計画を見せてくれる。

**何が嬉しいのか**

AIが見当違いの方向に進もうとしていたら、この段階で「それじゃなくて、こっちをやって」と軌道修正できる。

:::message
**ポイント**
Planning Mode（計画モード）で使う場合にのみ生成される。Fast Mode（高速モード）では省略される。モードの違いは後述する。
:::

### 2. 実装プラン --- 「どのファイルをどう変えるか」の設計図

タスクリストの次に表示される。「どのファイルを」「どう変更するか」の詳細な計画。

**何が嬉しいのか**

「そのファイルは触らないで」「この技術を使ってほしい」といったフィードバックを、コードが書かれる前に伝えられる。

**フィードバック方法**

実装プランの気になる箇所をクリックして、Google Docs のようにコメントを書き込める。

:::message alert
**注意**
コメントを書いたら必ず「送信」ボタンを押すこと。送信しないとAIに反映されない。
:::

### 3. スクリーンショット --- 変更前と変更後の比較画像

画面のデザインを変更した場合、変更前（Before）と変更後（After）の画像を自動で撮影して並べて見せてくれる。

**何が嬉しいのか**

コードを読めなくても、画像で「ちゃんと変わったか」を確認できる。

:::message
**非エンジニアに特に便利**
「ボタンの色を変えて」「文字を大きくして」といった見た目の変更は、スクリーンショットで一目で確認できる。コードを読む必要がない。
:::

### 4. ブラウザ録画 --- 操作の流れを動画で記録

AIがブラウザ上で行った操作（クリック、入力、ページ遷移など）を動画で記録してくれる。

**何が嬉しいのか**

自分でアプリを起動して確認しなくても、動画を見るだけで「ちゃんと動くか」が分かる。

:::message
**例え話**
テストパイロットが飛行機の試験飛行をしたあと、コックピットの映像を見せてくれるようなもの。「ちゃんと飛びましたよ」と言われるだけより、映像で確認できた方が安心。
:::

### 5. ウォークスルー --- 変更の要約と確認方法

AIがタスクを完了した後に生成される。「何を変えたか」「どうやって確認するか」をまとめた報告書。

**何が嬉しいのか**

変更の全体像を把握できる。ここにもコメントを書き込めるので、「テーマの色をオレンジに変えて」といった追加の指示も出せる。

:::message alert
**注意（繰り返し）**
ウォークスルーにコメントを書いた場合も、必ず「送信」ボタンを押すこと。書いただけでは送信されない。
:::

## 使い方の流れ

実際にArtifactsを使う流れを見てみよう。

1. Antigravity に指示を出す（例：「ログインページを作って」）
2. AIがタスクリストと実装プランを表示 → **確認して承認**
3. AIがコードを書く
4. スクリーンショットやブラウザ録画が生成される → **目で確認**
5. ウォークスルーが表示される → **問題なければ完了**

途中で気になる点があれば、Artifact 上にコメントを書いて「送信」を押すと、AIが修正してくれる。

:::message
**コメントの書き方**
Google Docs でコメントを付けるのと同じ感覚。気になる箇所をクリックして、テキストを入力し、「送信」を押すだけ。
:::

## 2つのモード

Antigravity には2つの動作モードがある。モードによって生成される Artifacts が変わる。

| モード | 説明 | Artifacts |
|--------|------|-----------|
| **Planning Mode** | 計画を立ててから作業する | 全て生成される |
| **Fast Mode** | すぐに作業に入る | 最小限（スクリーンショットのみ） |

:::message
**どちらを使うべき？**
「新しい機能を作る」「大きな変更をする」→ **Planning Mode がおすすめ**
「変数名を変える」「1行だけ修正する」→ Fast Mode で十分
迷ったら Planning Mode を選ぶのが安全。作業内容を事前に確認できるため。
:::

## Artifacts の確認方法

Artifacts は Antigravity の画面上で確認できる。

1. 「**Artifacts**」ボタンをクリック（Editor Viewでは画面右下、Agent Managerでは画面右上にある）
2. 生成された Artifacts が一覧で表示される
3. 各 Artifact をクリックして詳細を確認
4. 気になる箇所にコメントを追加（Google Docs と同じ感覚）
5. コメントを書いたら「**送信**」ボタンを押す

:::message alert
**送信ボタンを忘れずに**
コメントを書いても「送信」を押さないとAIには届かない。Google Docs のコメントと同様、書いた後に必ず送信すること。
:::

## Claude Code との違い

Antigravity と Claude Code は、どちらもAIにコードを書いてもらうツールだが、「作業結果の確認方法」が大きく異なる。

| 項目 | Antigravity | Claude Code |
|------|-------------|-------------|
| 作業の見える化 | Artifacts（画像・動画・タスクリスト） | テキストベースの報告 |
| フィードバック方法 | コメントを書き込む（Google Docs風） | チャットで指示する |
| 画面の変更確認 | スクリーンショットで自動比較 | 自分でブラウザを開いて確認 |
| 操作の検証 | ブラウザ録画で動画確認 | なし（自分でテスト） |
| 操作感 | GUI（画面をクリックして操作） | CLI（文字を入力して操作） |

:::message
**どちらが便利？**
画面の見た目を頻繁に確認したい場合は Artifacts のある Antigravity が便利。コードの変更内容を細かく制御したい場合は Claude Code が向いている。
:::

## 料金

Artifacts は Antigravity の標準機能。追加料金は不要。

| プラン | 料金 | Artifacts |
|--------|------|-----------|
| Free Preview | 無料 | 使える（利用制限あり） |
| Google AI Pro | 有料（2026年1月時点で$19.99/月程度） | 使える（制限が緩い） |

:::message alert
**注意：利用制限（ユーザー報告に基づく）**
無料プランでは2〜3時間の集中利用で制限に達することがある（公式には具体的な数値は公開されていない）。制限に達した場合は数時間待つ必要がある。残りの利用可能量を確認する手段がないため、大事な作業は早めに始めるのがよい。
:::

## まとめ

| Artifact | 何が分かるか | 非エンジニアへのおすすめ度 |
|----------|-------------|--------------------------|
| タスクリスト | AIの作業計画 | ★★★ |
| 実装プラン | 変更の設計図 | ★★☆ |
| スクリーンショット | 見た目の変更 | ★★★ |
| ブラウザ録画 | 操作の流れ | ★★★ |

特に非エンジニアにとって、コードを読まずに変更を確認できるスクリーンショットとブラウザ録画が便利。

「本当にちゃんと直ったの？」と不安に思ったら、Artifacts を確認しよう。
