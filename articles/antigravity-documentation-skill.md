## はじめに

ツールが完成した。動いた。嬉しい。

でも、次のハードルがあった。

---

「これ、どうやって使うの？」と聞かれた。

...えーと、ここをクリックして、ここに入力して...（口頭で説明）

---

「引き継ぎ資料作っておいて」と言われた。

...何を書けばいいの？READMEって何？

---

「来週の会議で報告して」と言われた。

...プレゼン資料も作るの？

---

非エンジニアの個人開発で四番目につらいのは、 **「作れたけど、説明できない」** こと。

本記事では、 **AIに質問に答えるだけで、引き継ぎ資料・プレゼン資料を自動生成するスキル** の詳細を解説する。

## 前提条件

- Google Antigravity インストール済み
  - まだの場合 → [Antigravityインストールガイド](antigravity-windows-install-guide)

## 解決策：ドキュメントスキル

### コンセプト

**「ドキュメント作って」と言うだけで、必要な資料が自動生成される。**

```
ユーザー: ドキュメント作って

AI: 誰向けの資料ですか？
    A. 自分用（備忘録）
    B. チームメンバー向け
    C. 外部の利用者向け
    D. 上司・経営層向け

ユーザー: D

AI: どの資料が必要ですか？
    ...
    E. プレゼン資料（上司報告用）

ユーザー: E

（5つの質問に答える）

AI: プレゼン資料を生成しました！
    docs/PRESENTATION.md を確認してください。
```

### 生成できる資料

| 資料 | 内容 | こんな時に |
|------|------|-----------|
| README.md | 使い方の説明 | 「どうやって使うの？」に答える |
| docs/SETUP.md | セットアップ手順 | 新しい人が使い始める時 |
| docs/USAGE.md | 詳しい使い方 | 機能が多い時 |
| docs/PRESENTATION.md | 上司報告用プレゼン | 会議で報告する時 |

### 5つの質問

| # | 質問 | 非エンジニア向け説明 |
|---|------|---------------------|
| 1 | 誰向け？ | 読む人に合わせて言葉遣いを変える |
| 2 | どの資料？ | 必要なものだけ作る |
| 3 | 読む人の詳しさは？ | 初心者向けか、詳しい人向けか |
| 4 | どのくらいの量？ | 簡潔に？詳しく？ |
| 5 | 追加で入れたいものは？ | FAQ、トラブル対応など |

## 実装方法

### Antigravityに指示を出す

```
skills/documentation/SKILL.md を作成してください。
フォルダがなければ作成してください。

【解決したい悩み】
非エンジニアの個人開発で、
1. 「どうやって使うの？」と聞かれて説明できない
2. 「引き継ぎ資料作って」と言われて何を書けばいいかわからない
3. 「会議で報告して」と言われてプレゼン資料が作れない

【自動参照する情報】
- 要件サマリー → プロジェクト概要
- docs/DESIGN.md → 使うツール
- docs/REVIEW.md → 注意点
- src/ → 使い方

【質問フロー（この順序を必ず守ること）】
Q0: 自動参照情報の確認（必須・最初に行う）
Q1: 誰向けのドキュメントですか？
    A. 自分用（備忘録）
    B. チームメンバー向け
    C. 外部の利用者向け
    D. 上司・経営層向け（非技術者）

Q2: どのドキュメントが必要ですか？
    A. README.md のみ（基本の使い方）
    B. README + セットアップ手順
    C. README + 詳しい使い方
    D. すべて
    E. プレゼン資料（上司報告用）← 会議で使える

Q3: 読む人はどのくらい詳しいですか？
    A. 全くの初心者（画面の操作から説明が必要）
    B. 少し分かる人（基本操作は知っている）
    C. 詳しい人（要点だけでOK）

Q4: どのくらいの量が必要ですか？
    A. 簡潔に（1-2ページ）← 忙しい上司向け
    B. 普通（3-5ページ）
    C. 詳しく（5ページ以上）

Q5: 追加で入れたいものはありますか？
    A. よくある質問（FAQ）
    B. 困った時の対処法
    C. 更新履歴
    D. 特になし

【プレゼン資料の構成】
1. タイトル（プロジェクト名、日付）
2. 課題（Before）— 今までの問題点
3. 解決策（After）— このツールで何が変わるか
4. 効果・数値 — 時間短縮、コスト削減など
5. 次のステップ — 今後の予定

【制約事項】
- 質問の省略禁止: Q0〜Q5 はすべて実行すること
- 順序変更禁止: 必ず Q0 → Q1 → Q2 → Q3 → Q4 → Q5 の順で実行
- 1問ずつ対話: 複数の質問を一度に聞かない
- 専門用語を使わない
- 画面のスクリーンショットを入れる場所を示す
```

## 非エンジニアの個人開発でよくあるドキュメントの悩み

| 悩み | このスキルでの解決 |
|------|-------------------|
| 何を書けばいいかわからない | AIが質問してくれる |
| READMEって何？ | 「使い方の説明」とわかりやすく表示 |
| プレゼン資料まで手が回らない | 5つの質問に答えるだけで生成 |
| 数字で効果を示せない | Before/After表を自動生成 |

## つまずきポイント

### 1. プレゼン資料の数字が出てこない

**問題**: 「効果」のセクションに具体的な数字が入らない。

**解決策**: 要件定義の時に「今まで○時間かかっていた」と伝えておく。または、この段階で「今まで月2時間かかっていた」と追加で伝える。

### 2. 前提ファイルがないと内容が薄い

**問題**: 要件サマリーや設計書がないと、プレゼン資料の内容が薄くなる。

**解決策**: ワークフローで順番に実行すれば、この段階では全ドキュメントが揃っている。

## まとめ

- **非エンジニアでもドキュメントが作れる** — AIが質問してくれる
- README、セットアップ手順、 **プレゼン資料** まで自動生成
- 上司への報告資料もBefore/After形式で作れる
- 「今まで○時間→○分に短縮」のような数字も自動で入る
