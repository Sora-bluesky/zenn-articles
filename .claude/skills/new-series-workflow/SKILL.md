# 新シリーズ作成ワークフロー

新しい記事シリーズを立ち上げる際のワークフロー。
公式ベストプラクティスを解説するシリーズなど、体系的な記事群を作成する時に使用する。

## 起動条件

以下のような依頼があった場合に自動適用：
- 「新しいシリーズを作りたい」
- 「〇〇について体系的に解説する記事を作りたい」
- 「公式ベストプラクティスを解説するシリーズ」

## ワークフロー

### Phase 1: Deep Research

公式ドキュメント・ベストプラクティスを調査する。

**参照先**：
- Claude Code: https://code.claude.com/docs/en/best-practices
- Anthropic Blog: https://www.anthropic.com/engineering/claude-code-best-practices
- Google Antigravity: https://antigravity.google/docs

**出力**：
- 全体構成（セクション、トピック一覧）
- 各トピックの優先度（最高/高/中/低）
- 核心メッセージの抽出

### Phase 2: メタ認知レビュー（3回）

#### 第1回：対象読者の定義
- 誰向けのシリーズか（初心者/中級者/上級者）
- 既存シリーズとの棲み分け
- ターゲット層のミスマッチがないか

#### 第2回：構成の検討
- 全体で何部が適切か（5部以下推奨）
- 各記事が独立して価値を持つか
- 読者が「読み切れる」量か

#### 第3回：重複チェック
- 既存記事との内容重複がないか
- 重複がある場合、観点の違いで棲み分けられるか
  - 既存：機能の紹介（How）
  - 新規：ベストプラクティスの理解（Why & When）

### Phase 3: 構成決定

ユーザーに以下を提示して承認を得る：

```markdown
## 推奨シリーズ構成：全[N]部

| # | 記事タイトル | 概要 |
|---|-------------|------|
| 1 | [タイトル] | [概要] |
| 2 | [タイトル] | [概要] |
| ... | ... | ... |

### 発展編（将来）
- [追加候補1]
- [追加候補2]
```

### Phase 4: 記事作成

承認後、各記事を作成する。

**タイトル規約**（CLAUDE.md参照）：
- Claude Code: `Claude Code [カテゴリ]：[ベネフィット]`
- 実践Tips: `Claude Code 実践Tips [#]：[テーマ]`
- Google Antigravity: `Google Antigravity [スキル名]スキル：[ベネフィット]`

**記事構成**：
- シリーズ構成ブロック（冒頭）
- 本文
- 次の記事への誘導
- 参考リンク（公式ドキュメント）
- 関連記事

**公開設定**：
- 初回は `published: false`（下書き）でプッシュ
- レビュー後に `published: true` に変更
